# Sound Vault — Graphite Design System

**Date:** 2026-05-25
**Status:** Approved direction (selected from `ui-mockups-skeuo-2026-05-25.html` Variant C)
**Target:** PySide6 desktop app at `product/sound-vault-desktop/src/sound_vault/ui/`
**Audience:** Implementation agent porting the visual language from HTML/CSS into QSS + QPainter

---

## 1. North Star

A tactile retro-futurist control-room dashboard in the lineage of **Mac OS X Tiger / Leopard Pro apps** — Logic Pro 7/8, Aperture 1.x, early Final Cut Studio. Heavier brushed-metal chrome than consumer iTunes, restrained color, dark glowing HUDs for readouts, machined-aluminum tabs and buttons, professional density and quiet authority.

This is **not** glossy iTunes, **not** Winamp green-LCD novelty, **not** MySpace maximalism. It is the brushed-metal pro workstation.

**Mood words:** machined, graphite, restrained, instrument-panel, evidence-first, professional, quietly luminous.

**Forbid:** flat SaaS cards, soft pastel shadows, neon glow, cyberpunk, glassmorphism without bevel, Material elevation tiers, accent gradients that look generated by a tool.

---

## 2. Light & Material Rules

Three rules everything in the app obeys.

1. **Light comes from the top.** Every surface has a brighter top edge (~1px highlight) and a darker bottom edge (~1px shadow). Inset wells reverse this: dark on top, light on bottom.
2. **Metals are textured, HUDs are inset and glowing.** Aluminum/graphite surfaces always carry a faint horizontal brushing. Dark HUDs are recessed wells with a soft blue inner glow and shadowed depth.
3. **Color is a status signal, never decoration.** The palette is graphite + aluminum + black. Accent color (cool blue) is reserved for selection, active state, and HUD text. Status colors (amber, red, green) are reserved for archive-health meters and severity dots.

**QSS notes:**
- Use `font-family: "Helvetica Neue", "Helvetica", "Arial", sans-serif;` at the QApplication level.
- Use `font-family: "SF Mono", "Menlo", "Consolas", monospace;` on HUD widgets.
- `text-shadow` does not exist in QSS. To get the HUD blue glow, use either:
  - a `QGraphicsDropShadowEffect` with `color=QColor(120,180,255,140)`, `blurRadius=8`, `offset=(0,0)`, OR
  - a custom `QLabel` subclass that overrides `paintEvent` and draws the text with a soft glow.
- Top-edge highlight text (`text-shadow: 0 1px 0 rgba(255,255,255,0.5)`) on aluminum labels: do it with a 1px-offset white shadow via `QGraphicsDropShadowEffect`, or accept that QSS can't do it and let the brushed-metal background carry the depth on its own.

## 5. Surface Recipes

These are the load-bearing material recipes. Reuse them everywhere — don't invent new gradients per component.

### 5.1 Brushed Aluminum (title bar, tab bar, transport buttons)

**Visual:** A warm metallic surface with very fine horizontal brushing, a brighter top edge, a darker bottom edge.

**CSS source (Variant C title bar):**
```css
background:
  repeating-linear-gradient(0deg, rgba(255,255,255,0.07) 0 1px, transparent 1px 2px),
  repeating-linear-gradient(90deg, rgba(0,0,0,0.025) 0 1px, transparent 1px 2px),
  linear-gradient(180deg, #b8b6ae 0%, #888578 60%, #6f6c60 100%);
border-bottom: 1px solid #2a2820;
box-shadow:
  inset 0 1px 0 rgba(255,255,255,0.55),
  inset 0 -1px 0 rgba(0,0,0,0.45);
```

**QSS port:**
QSS does NOT support `repeating-linear-gradient`. Two acceptable paths:

**Option A (preferred): pre-bake a brushed-metal PNG tile** (e.g. 2px × 64px) and apply via `background-image: url(...); background-repeat: repeat-x;` layered over the QSS `qlineargradient`. Bake the texture once at high res, embed via Qt resource system.

**Option B: custom `BrushedMetalWidget(QWidget)` with `paintEvent`.** Paint order:
1. `QLinearGradient` (top→bottom) — base color stops.
2. Loop horizontal 1px lines at 2px spacing in `rgba(255,255,255,18)` — brushing.
3. Optional: `QImage` of pre-baked low-amplitude noise, painted at 0.04 opacity for grain.
4. Draw 1px top highlight in `rgba(255,255,255,140)`.
5. Draw 1px bottom shadow in `rgba(0,0,0,115)`.
6. Draw 1px hard border in `#2a2820`.

For most of the app, **Option A is fine** and avoids 12 custom widget classes. Reserve Option B for the main title bar and the main deck background where the texture really sells the surface.

### 5.2 Graphite (transport bar, sidebar gradient base, inspector)

**Visual:** Darker than the aluminum chrome. Vertical fade from medium graphite to near-black. Optional faint vertical brushing.

**CSS source (Variant C transport):**
```css
background:
  repeating-linear-gradient(90deg, rgba(255,255,255,0.04) 0 1px, transparent 1px 2px),
  linear-gradient(180deg, #4a4a48 0%, #2c2c2a 100%);
border-bottom: 1px solid #1a1a18;
box-shadow:
  inset 0 1px 0 rgba(255,255,255,0.18),
  inset 0 -1px 0 rgba(0,0,0,0.7);
```

**QSS port:**
```css
QWidget#transport {
  background: qlineargradient(x1:0, y1:0, x2:0, y2:1,
    stop:0 #4a4a48, stop:1 #2c2c2a);
  border-bottom: 1px solid #1a1a18;
  border-top: 1px solid rgba(255,255,255,46);
}
```
Add the vertical brushing tile via background-image if desired. The `inset 0 -1px 0 rgba(0,0,0,0.7)` shadow has no QSS equivalent — let the hard `border-bottom` carry it.

### 5.3 Dark HUD (timecode display, archive-health panel, inspector HUD)

**Visual:** A deeply recessed black-blue well that looks lit from inside with soft blue light.

**CSS source (Variant C transport HUD):**
```css
background:
  radial-gradient(ellipse at 50% 0%, rgba(120,180,255,0.08) 0%, transparent 60%),
  linear-gradient(180deg, #0a0c10 0%, #1a1e26 100%);
border: 1px solid #000;
border-radius: 6px;
box-shadow:
  inset 0 2px 4px rgba(0,0,0,0.9),
  inset 0 1px 0 rgba(120,180,255,0.12),
  0 1px 0 rgba(255,255,255,0.15);
```

**QSS port:**
```css
QFrame#hud {
  background: qlineargradient(x1:0, y1:0, x2:0, y2:1,
    stop:0 #0a0c10, stop:1 #1a1e26);
  border: 1px solid #000;
  border-radius: 6px;
}
```
The radial blue-glow overlay and inset shadow cannot be expressed in QSS. Two options:
- Accept the loss — the dark gradient + black border + blue HUD text glow is 80% of the effect.
- Override `paintEvent` to draw the radial gradient overlay and inset shadow manually.

For the **archive-health panel** in the sidebar (see §6.6), the dark HUD recipe is mandatory — it should feel like a separate instrument embedded in the sidebar.

### 5.4 Raised Transport Button (machined aluminum pill)

**CSS source:**
```css
width: 32px; height: 30px; border-radius: 6px;
background: linear-gradient(180deg, #d8d6cc 0%, #a8a496 50%, #7c7868 100%);
border: 1px solid #1a1a14;
box-shadow:
  inset 0 1px 0 rgba(255,255,255,0.8),
  inset 0 -1px 1px rgba(0,0,0,0.25),
  0 1px 0 rgba(0,0,0,0.6);
color: #1a1a1a; font-size: 11px;
text-shadow: 0 1px 0 rgba(255,255,255,0.4);
```

**QSS port:**
```css
QToolButton#transportBtn {
  min-width: 32px; min-height: 30px;
  border-radius: 6px;
  background: qlineargradient(x1:0, y1:0, x2:0, y2:1,
    stop:0 #d8d6cc, stop:0.5 #a8a496, stop:1 #7c7868);
  border: 1px solid #1a1a14;
  color: #1a1a1a;
}
QToolButton#transportBtn:hover {
  background: qlineargradient(x1:0, y1:0, x2:0, y2:1,
    stop:0 #e6e4d8, stop:0.5 #b8b4a4, stop:1 #8c8878);

QToolButton#transportBtn:pressed {
  background: qlineargradient(x1:0, y1:0, x2:0, y2:1,
    stop:0 #7c7868, stop:0.5 #a8a496, stop:1 #d8d6cc);
  border-color: #000;
}
```
**Pressed state inverts the gradient** — that's the tactile move. Don't skip it.

## 6. Component Specs

### 6.1 Title Bar (46px tall)

- Brushed aluminum surface (recipe 5.1)
- Traffic lights (13px dia, red/yellow/green, see §3) — **decorative on macOS native window, real on custom frameless windows**
- Tab bar inline: 4 tabs, 26px tall, machined aluminum (default) vs. dark-blue graphite (selected, recipe matches §3 `blue-tab-*`)
- Right-aligned app title in 12px Helvetica Neue 700 with white 1px top-edge text shadow

### 6.2 Transport Bar (74px tall)

Layout (left to right):
1. Transport button row: ⏮ ▶ ⏭ — three 32×30 machined buttons (5.4)
2. Center HUD: dark well (5.3), contains a 2-column grid — left column "now playing" block (title 13px, sub 10px blue-dim), right column large monospace timecode (22px, blue-glow)
3. Status capsule pill (5.5) — e.g. "● IDLE" or "● RUNNING"
4. Search pinstripe field (5.6), 200px wide

The HUD is the **heart** of the transport. It should be ~50px tall, recessed, glowing softly. Do not flatten it.

### 6.3 Tab Bar (in title bar)

- 4 tabs: LIBRARY · EVIDENCE · TRANSCRIPTS · POPULARITY
- 26px tall, no gap between adjacent tabs (1px hairline)
- Default: machined aluminum, dark text with white top-shadow
- Selected: dark blue-graphite gradient (`#5a6878` → `#364452`), white text with dark top-shadow, slightly darker border

These replace the current button-based "portal tabs" in `desktop.py` and should behave like real tabs (mutually exclusive, content-switching).

### 6.4 Sidebar (212px wide)

### 6.6 Archive Health Panel (sidebar bottom)

This is one of the highest-leverage moves in this design. Replace the current static "Worker / idle" card with a live HUD.

```
┌─ ARCHIVE HEALTH ─────────────┐
│ ▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓░░  92% │  AUDIO       (blue bar)
│ ▓▓▓▓▓▓▓▓▓▓▓▓▓░░░░░░░░░  64% │  ARTWORK     (amber bar)
│ ▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓░░░░░░░  78% │  TRANSCRIPT  (blue bar)
│ ▓▓▓▓▓▓░░░░░░░░░░░░░░░░  34% │  VIDEOS      (red bar)
└──────────────────────────────┘
```

- Container: dark HUD recipe (5.3), 10px padding, 5px border-radius
- Title: 9px uppercase 0.16em `#6e8aae`, margin-bottom 6px, SF Mono
- Each row: 6px-tall bar + label/value row underneath
- Bar:
  - Track: `rgba(0,0,0,0.6)`, 3px border-radius, inset 1px black shadow
  - Fill: vertical gradient by health tier:
    - ≥80%: `#a0d0ff` → `#4a7ac0` (blue-cool — reads as "healthy")
    - 50–79%: `#ffd87a` → `#d09810` (amber)
    - <50%: `#ff8a7a` → `#c43820` (red)

Implement as a custom `ArchiveHealthPanel(QFrame)` widget. Data source: query `IndexDatabase` for `total_records`, `with_audio`, `with_artwork`, `with_transcript`, `with_videos`. Recompute on every successful index rebuild.

## 7. State Variants

| State | Treatment |
|---|---|
| **Hover** on raised metal button | Lighten top gradient stop by ~10% |
| **Pressed** on raised metal button | **Invert the gradient** (top↔bottom swap) and darken border |
| **Disabled** on raised metal button | Desaturate gradient to grayscale, drop opacity to 0.5 |
| **Selected** row/tab/sidebar item | Switch to blue accent recipe (§3) |
| **Focus** on text input | 1px `#4a7ac0` border replaces black border; do not glow |
| **Error/destructive** button | Red gradient `#c44818` → `#7a2408` (Variant D's orange-red — reuse only here) |

## 8. Don'ts

- **No drop shadows under arbitrary panels.** Depth comes from inset wells, borders, and bevels. The whole window has one outer drop shadow; nothing else does.
- **No animation longer than 120ms.** Tactile UIs feel mechanical, not springy.
- **No gradient on text.** Text is solid color with at most a 1px top highlight or bottom shadow.
- **No rounded corners on the table or sidebar.** Only buttons, pills, HUDs, the artwork well, and the outer window get radius.
- **No more than one accent color per region.** Blue is the only accent in the chrome. Amber/red appear only inside the health meters and severity dots.
- **No emoji as iconography.** The mockup uses Unicode glyphs as placeholders. Replace with a small SVG icon set (♫ → music note, ★ → favorite, ⌂ → home, ⎘ → review queue, ⊟ → dedupe, ⚙ → gear, ▶ → play, ⏭ → next, ⏮ → prev).

## 11. Acceptance — what "done" looks like

A reviewer should look at the app and say:

- "That looks like a Pro app from 2007, not a SaaS dashboard."
- "The transport HUD looks lit from inside."
- "The sidebar health panel tells me at a glance what's missing in my vault."
- "Buttons feel pressable — pressing one inverts its gradient."
