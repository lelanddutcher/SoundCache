# Sound Vault — Gunmetal Design System

**Status:** Active. Use this as the single source of truth for all UI work.
**Supersedes:** `design-system-graphite-2026-05-25.md` (kept for context but its tokens are out of date).
**Implementation lives at:** `src/sound_vault/ui/desktop.py` — `STYLESHEET` constant + a handful of custom widget classes (`StatusDot`, `HealthBar`, `ArchiveHealthPanel`).

---

## 1. Identity

A tactile retro-futurist control-room dashboard in the lineage of Mac OS X Tiger / Logic Pro / Aperture 1.x — but cooler and more industrial. **Gunmetal**, not graphite: a blue-leaning gray that reads as machined steel under a soft cool light. Surfaces are layered (brushed chrome over inset HUDs over deep graphite shells). Color is a status signal, not decoration. Typography is restrained — small, dense, uppercase where structural.

**Forbid:** flat SaaS cards · cyberpunk neon · Material elevation tiers · soft pastel shadows · drop shadows under arbitrary panels · glassmorphism without a real bevel · gradient text · emoji as iconography · animations longer than 120 ms · accent gradients that look generated by a tool.

---

## 2. Three Load-Bearing Rules

Every surface in the app obeys these. New components MUST as well.

1. **Light from the top.** Every raised surface has a brighter top edge (~1px highlight) and a darker bottom edge (~1px shadow). Inset surfaces reverse it: dark top, light bottom.
2. **Metals are layered gradients, HUDs are recessed wells.** Aluminum/steel surfaces use 3–4 stop linear gradients to feel machined, not painted. Dark HUDs use deep blue-black with a faint top-edge blue rim to look "lit from inside."
3. **Color is a status signal.** The palette is gunmetal + steel + black. Cool blue is the *only* accent (selection, active, HUD text, primary CTA). Amber/red/green appear ONLY inside health meters and severity dots.

## 5. Surface Recipes (use these; don't invent new gradients)

### 5.1 Brushed Steel Chrome (`#chromeHeader`)

```css
background: qlineargradient(x1:0, y1:0, x2:0, y2:1,
    stop:0 #b8bec4, stop:0.4 #9098a0, stop:0.6 #7a8088, stop:1 #5a6068);
border-top: 1px solid #d4d8dc;       /* highlight */
border-bottom: 1px solid #1a1e22;    /* shadow */
border-left: 1px solid #5a6068;
border-right: 1px solid #1a1e22;
border-radius: 8px;
```

The 4-stop gradient with mid-point inflection is what gives the "machined" feel. 2-stop gradients read painted; 4-stop reads machined.

### 5.2 Transport Deck Graphite (`#transportDeck`)

```css
background: qlineargradient(x1:0, y1:0, x2:0, y2:1,
    stop:0 #424850, stop:0.5 #30363c, stop:1 #1c2026);
border-top: 1px solid #5c6068;
border-bottom: 1px solid #0c0e12;
border-left: 1px solid #0c0e12;
border-right: 1px solid #0c0e12;
border-radius: 6px;
```

### 5.3 Machined Steel Button (`#transportButton`, generic `QPushButton`)

```css
background: qlineargradient(x1:0, y1:0, x2:0, y2:1,
    stop:0 #d4d8dc, stop:0.45 #a4a8ac, stop:0.55 #8a8e92, stop:1 #6a6e72);
border: 1px solid #0c0e12;
border-radius: 4-6px;
color: #14181c;
```

Hover lightens every stop by ~10%. **Pressed INVERTS the gradient** (top↔bottom swap) and darkens border — this is the tactile mechanical move; don't skip it.

### 5.4 Dark HUD Well (`#capsuleDisplay`, `#archiveHealthPanel`, `QTextEdit`, `#artwork`, `#preview` for sidebar HUD area)

```css
background: qlineargradient(x1:0, y1:0, x2:0, y2:1,
    stop:0 #06080c, stop:0.06 #0a0e14, stop:1 #181e26);
border: 1px solid #000000;
border-top: 1px solid #2a3848;   /* the blue rim — "lit from inside" */
border-radius: 4-6px;
```

The 0.06 second-stop creates a darker depression near the top. The `border-top: 1px solid #2a3848` (the **blue rim**) is critical — it's the single detail that makes a dark rectangle read as "lit." If you build a new HUD-style surface and skip the blue rim, it looks dead.

### 5.5 Inset Card (`#statCard`, `#pairingCard`, `QGroupBox`)

```css
background: qlineargradient(x1:0, y1:0, x2:0, y2:1,
    stop:0 #1c2026, stop:0.04 #181c22, stop:1 #0e1216);
border: 1px solid #02040a;
border-top: 1px solid #2a3038;
border-radius: 5px;
padding: 10px;
```

Same trick: a thin lighter `border-top` simulates an inner-top highlight.

### 5.6 Selected Row / Active Nav (cool blue)

```css
background: qlineargradient(x1:0, y1:0, x2:0, y2:1,
    stop:0 #2c405e, stop:0.04 #2a3e5a, stop:1 #14223a);
color: #e8eef8;
border: 1px solid #4a7ac0;
border-top: 1px solid #6a92d2;
```

### 5.7 Primary CTA Button (`#primaryButton`)

```css
background: qlineargradient(x1:0, y1:0, x2:0, y2:1,
    stop:0 #5a708a, stop:0.5 #3a4a5e, stop:1 #1e2a3a);
color: #e8eef8;
border: 1px solid #0c1218;
border-top: 1px solid #8aa4c8;
```

Same shape as a steel button but in the cool-blue family. Reserved for the single most important action in a view (currently: Rebuild Index).

### 5.8 Destructive Button (`#dangerButton`)

```css
background: qlineargradient(x1:0, y1:0, x2:0, y2:1,
    stop:0 #d8542a, stop:0.5 #aa3814, stop:1 #6c2008);
color: #ffe8d8;
border: 1px solid #3a1004;
border-top: 1px solid #f47040;
```

Only for destructive operations. Don't reuse orange anywhere else.

## 6. Component Patterns

### 6.1 Status LED (`StatusDot` widget — `desktop.py`)

A custom `QWidget(setFixedSize(12,12))` with `paintEvent`. Four states: `online` (green), `running` (amber), `error` (red), `waiting` (cool blue). Renders as a colored gradient orb with a top-left specular highlight and a soft glow ring on active states.

```python
self.index_status_dot = StatusDot()
self.index_status_dot.set_state("online")
```

**When to use:** index status, worker status, anywhere you want a tactile colored LED. Don't use a colored `●` text character — it lacks depth.

### 6.2 Health Bar (`HealthBar` widget)

A 6px-tall `QPainter`-drawn bar with auto tier-coloring: blue ≥80%, amber ≥50%, red <50%. Background is a recessed black track, fill is a vertical gradient with a 1px top highlight.

```python
bar = HealthBar()
bar.set_percent(0.92)  # 92% → blue fill
```

### 6.3 Archive Health Panel (`ArchiveHealthPanel`)

Self-contained `QFrame` with 4 `HealthBar`s (audio/artwork/transcript/videos) + labels. Updated via:

```python
panel.update_from_counts(db.archive_health_counts())
```

### 6.4 Now Playing Capsule (the dark HUD in the transport)

`QFrame` with:
- `setFixedHeight(56)` — never grows
- `setMinimumWidth(280)` — never collapses below this
- Eyebrow label ("NOW PLAYING") in `#displayEyebrow` style
- Title label in `#displayTitle` style with `setSizePolicy(QSizePolicy.Policy.Ignored, ...)` so long song titles get elided (~48 chars max in our code) instead of pushing the capsule wider

### 6.5 Library Table (`SoundTableWidget`, subclass of `QTableWidget`)

Default visible columns on a fresh install: ★ favorite, play, sound, artist/source, status, packaged, popularity, videos. Hidden by default: added, local audio, trend/context. User preferences persist via `settings.set_hidden_table_columns(...)`.

Custom delegates:
- `FavoriteButtonDelegate` renders ★/☆ as a tappable raised button via `QStyleOptionButton`
- `PlayButtonDelegate` renders ▶/— as a tappable raised button

**Perf rules:**
- `refresh_table` uses `setUpdatesEnabled(False)` + `blockSignals(True)` + caches `favorite_music_ids()` ONCE per refresh (don't re-query inside the row loop).
- Sort header clicks defer the actual rebuild via `QTimer.singleShot(0, refresh_table)` so the indicator updates instantly.

### 6.6 Inspector (`#preview` frame)

Trimmed layout: artwork → title → scrubber → time → action row (Copy / TikTok / Folder) → Transcript HUD. Other widgets (`preview_meta`, `preview_tags`, `playback_status`, evidence group, video group) are constructed and parented to `scroll_body` but NOT added to the layout. Existing setters still work; they just aren't rendered. **When you add new widgets that should be hidden by default, always set a parent — orphan `QWidget`s get GC'd by Qt's C++ side.**

### 6.7 Sidebar Nav Items (`#navButton`)

`QPushButton` with `setObjectName("navButton")`. Active state via a custom property:

```python
btn.setProperty("active", "true")
btn.style().unpolish(btn); btn.style().polish(btn)
```

Same pattern for `#libraryBinButton`. The `dropHover` property is used during drag-drop to highlight the bin target.

## 7. Interaction States

| State | Treatment |
|---|---|
| Default | Per recipe in §5 |
| Hover (raised metal) | Lighten every gradient stop by ~10% |
| Pressed (raised metal) | **Invert the gradient** + darken border |
| Hover (dark/inset) | Shift to the cool-blue family (top stop `#243248` → `#142030`), set border to `#2a4060`. Same shape as selection but with thinner border. |
| Selected | Cool-blue selection recipe (§5.6) |
| Focused (input) | Replace black border with `#4a7ac0`, add `#6a92d2` top edge |
| Disabled | Desaturate to flat `#4a4e54`→`#383c42`, text `#6a6e74`, border `#2a2e34` |
| Drop target | Same as selected, but border `#4a7ac0` becomes more prominent |

## 8. Layout Rules

- **Window minimum:** 900×600. Don't break below this.
- **Sidebar:** min 200 / max 320 (it's flexible, not fixed). The sidebar's purpose is nav + Archive Health HUD; long bin names should not push it past 320.
- **Inspector:** min 330 / max 560. Same logic.
- **Artwork well:** `setFixedSize(210, 210)`. Square, never grows. Inspector min-width is set so it always fits with breathing room.
- **Now Playing capsule:** `setFixedHeight(56)`, `setMinimumWidth(280)`. Title elides; capsule never changes size between selections.
- **Title row (above transport):** intentionally restrained — 18px title, 11px compact path (last 3 segments with full path in tooltip).

## 9. QSS / PySide6 Practicalities

### Supported in QSS

- `qlineargradient(...)`, `qradialgradient(...)`, `qconicalgradient(...)` — Qt's gradient syntax
- Per-side borders with different colors (this is how we fake bevels)
- `border-radius`, `padding`, `margin`
- `text-transform: uppercase` (Qt 6+)
- `letter-spacing` (Qt 6+)
- Pseudo-states: `:hover`, `:pressed`, `:focus`, `:checked`, `:disabled`
- Pseudo-elements on subclassable widgets: `::item`, `::handle`, `::groove`, `::section`, etc.
- Property selectors: `QPushButton[active="true"]`

### NOT supported

- `box-shadow` — use `QGraphicsDropShadowEffect` or `paintEvent`
- `text-shadow` — same
- `backdrop-filter`, `filter` — none of it
- Multiple `background-image` layers via comma — Qt picks one
- `aspect-ratio` — use `setFixedSize` or `sizeHint` override
- Repeating background patterns from inline data URLs — work for some Qt versions, unreliable. If you need a tile, ship a PNG via Qt resource or `paintEvent`
- CSS variables — there are none; if you want tokens, define Python constants and string-format the stylesheet (we currently don't bother)

### When to write a custom widget

Reach for `paintEvent` when:
- You need a glow (status LED)
- You need a gradient that ISN'T linear/radial/conical (a real radial bump, brushed texture)
- You need pixel-precise composition that QSS can't express (e.g., the inner highlight inside a HUD)
- The thing is animated

Use the `StatusDot` and `HealthBar` classes in `desktop.py` as templates.

### Naming convention

- Existing object names are load-bearing. Don't rename them; the stylesheet pins on them.
- New object names: lowerCamelCase (`#archiveHealthPanel`, `#capsuleDisplay`, `#rowPlayButton`).
- Reserve `Q*` selector prefixes for default Qt widget classes.

## 11. Anti-Patterns (real things that have gone wrong here)

1. **Constructing a `QLabel` with no parent and not adding it to a layout.** Qt's C++ side GC's it. Symptom: `RuntimeError: Internal C++ object already deleted`. Fix: always pass a parent or addWidget to a layout that's installed on a real container.
2. **Per-row file reads in the table.** `vm.is_favorite(music_id)` was reading `library_collections.json` from disk per row → 2000+ disk reads per refresh. Always pull set/collection state ONCE outside the row loop, use set membership inside.
3. **`.exists()` on every catalog path in summary mode.** Iteration 7 trusted the catalog paths. The catalog had stale `/nas/` prefixes from the user's old mount. Iteration 7b reintroduced rebase: `folder / Path(value).name`. Don't validate paths in summary mode; rebase against the resolved folder instead.
4. **A staticmethod that hits the filesystem called per row.** `vm.play_target_for(record)` does `.exists()`. Don't use it for "is this playable" predicates in row scans — use `_record_has_playable_pointer` (pure Python).
5. **Stylesheet references to deleted attributes.** When trimming the inspector, removing `self.preview_json` left a `self.preview_json.setPlainText(...)` site that crashed on first selection. Always grep for every reference when you remove a widget.
6. **Variable-width labels in fixed-rhythm regions.** The Now Playing title used to expand with text. Set `setSizePolicy(QSizePolicy.Policy.Ignored, ...)` and elide.
7. **The plain `●` character as a status indicator.** Looks dead. Use `StatusDot`.
8. **Hover state set only via QSS background change.** That alone reads "highlight" — combine background change with border-color change for depth.
9. **`text-shadow` in QSS.** Doesn't render. Either drop it or use `QGraphicsDropShadowEffect`.

## 12. Future Polish Pending

When more time is available, in priority order:

1. **`QAbstractTableModel` migration** — replace `QTableWidget` with `QTableView` + custom model. ~10× faster refreshes, sort/filter become O(model) instead of O(all rows). Touches `desktop.py` library table construction, both delegates, `refresh_table`, `_sort_library_rows`, drag/drop, selection. Several-hundred-line change. Single biggest perceived-perf win available.
2. **Real brushed-metal texture** — ship a 64×64 PNG noise tile via Qt resources (`.qrc`), reference from QSS as `background-image: url(:/brushed.png)`. Currently we fake it with gradient stops; texture would push the chrome from "very good" to "convincing."
3. **SVG icon set** — see §10.
4. **A/B duplicate-review layout** — two-up candidate cards with waveform + transcript diff + keeper toggle.
5. **Hardware-style filter pills** — replace `QComboBox` filters with capsule pill-toggle groups (already drafted in the original brainstorm). Won't fit the existing layout without inspector and table shrinks.
6. **Animation polish** — when something appears or changes state, a 80–120ms ease-out. Currently everything is instant, which is fine but lacks craft. Use `QPropertyAnimation` sparingly.

## 13. Quick Reference for New Components

Building anything new? Walk this checklist:

1. **Is it a raised surface?** → Use the steel/aluminum gradient (§5.3) with 4 stops. Add the inner top highlight (1px brighter border-top). Bottom shadow border.
2. **Is it a dark inset surface?** → Use the HUD recipe (§5.4). **Include the blue rim** (`border-top: 1px solid #2a3848`).
3. **Is it text on a dark surface?** → `#c6cdd4` body, `#e6ecf2` title, `#7a8290` muted. Mono if it's a HUD readout, otherwise Helvetica Neue.
4. **Does it have a state?** → Hover, pressed, active variants per §7. Don't ship a button without a pressed state.
5. **Does it indicate status?** → Use the `StatusDot` widget or a `HealthBar`. Don't invent a new colored shape.
6. **Does it need to never shift the layout?** → `setFixedHeight` or `setFixedSize`, set sizePolicy on inner labels to `Ignored` horizontally, elide long text in code.
7. **Did you parent every widget?** → Pass `parent=<container>` to the constructor. Never construct a `QWidget` with no parent unless you immediately `addWidget` it to a layout that owns it.
8. **Did you smoke-test row selection?** → If the inspector or table is involved, run an offscreen `QApplication` test that constructs the window, runs `rebuild_index()`, calls `table.selectRow(0)`, and asserts no exception. The crashes in §11 would have been caught by this.
